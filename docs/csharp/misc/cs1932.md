# <a name="compiler-error-cs1932"></a><span data-ttu-id="aeb01-103">编译器错误 CS1932</span><span class="sxs-lookup"><span data-stu-id="aeb01-103">Compiler Error CS1932</span></span>

<span data-ttu-id="aeb01-104">无法将“expression”赋值给范围变量</span><span class="sxs-lookup"><span data-stu-id="aeb01-104">Cannot assign 'expression' to a range variable.</span></span>  
  
 <span data-ttu-id="aeb01-105">编译器必须能够推断出的某个范围变量的类型，不管其是在 `from` 子句中引入还是在 `let` 子句中引入。</span><span class="sxs-lookup"><span data-stu-id="aeb01-105">The compiler must be able to infer the type of a range variable, whether it is introduced in a `from` clause or a `let` clause.</span></span> <span data-ttu-id="aeb01-106">其不能为 null，因为 null 不是一种类型，而且不能使用不安全类型的表达式对其赋值。</span><span class="sxs-lookup"><span data-stu-id="aeb01-106">It cannot be null because null is not a type, and it cannot be assigned with an expression of an unsafe type.</span></span>

## <a name="example"></a><span data-ttu-id="aeb01-110">示例</span><span class="sxs-lookup"><span data-stu-id="aeb01-110">Example</span></span>  

 <span data-ttu-id="aeb01-111">下面的代码会生成 CS1932，因为不能推断范围变量的类型。</span><span class="sxs-lookup"><span data-stu-id="aeb01-111">The following code generates CS1932 because the type of the range variable cannot be inferred.</span></span> <span data-ttu-id="aeb01-112">将值转换为预期类型以修复此错误，如下面的示例所示。</span><span class="sxs-lookup"><span data-stu-id="aeb01-112">Cast the value to the intended type to fix the error, as shown in the following example.</span></span>  
  
```csharp  
// CS1932.cs  
using System.Linq;  
class Test  
{  
    static void Main()  
    {  
  
        var x = from i in Enumerable.Range(1, 100)  
                let k = null // CS1932  
                // Try the following line instead.  
                let k = (string) null  
                select i;  
    }  
}  
```
