## <a name="principle-of-least-privilege"></a><span data-ttu-id="b9ce6-130">最低权限原则</span><span class="sxs-lookup"><span data-stu-id="b9ce6-130">Principle of least privilege</span></span>

<span data-ttu-id="b9ce6-131">计算机安全的一项初步理念是 (POLP) 的最小特权原则。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-131">One of the founding ideas in computer security is the Principle of Least Privilege (POLP).</span></span> <span data-ttu-id="b9ce6-132">它实际上是一个基本概念，其中的大多数形式都是数字或物理。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-132">It's actually a foundational idea in most any form of security be it digital or physical.</span></span> <span data-ttu-id="b9ce6-133">简而言之，原则是，任何用户或进程都应具有执行其任务所需的最少权限。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-133">In short, the principle is that any user or process should have the smallest number of rights possible to execute its task.</span></span>

<span data-ttu-id="b9ce6-134">例如，在银行 tellers：访问 safe 是一种不常见的活动。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-134">As an example, think of the tellers at a bank: accessing the safe is an uncommon activity.</span></span> <span data-ttu-id="b9ce6-135">因此，平均出纳不能打开安全的。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-135">So, the average teller can't open the safe themselves.</span></span> <span data-ttu-id="b9ce6-136">若要获取访问权限，他们需要通过银行经理来升级其请求，该管理人员执行额外的安全检查。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-136">To gain access, they need to escalate their request through a bank manager, who performs additional security checks.</span></span>

<span data-ttu-id="b9ce6-137">在计算机系统中，有一个极好的示例是用户连接到数据库的权限。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-137">In a computer system, a fantastic example is the rights of a user connecting to a database.</span></span> <span data-ttu-id="b9ce6-138">在许多情况下，只需使用单个用户帐户即可生成数据库结构并运行应用程序。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-138">In many cases, there's a single user account used to both build the database structure and run the application.</span></span> <span data-ttu-id="b9ce6-139">除极端情况外，运行应用程序的帐户不需要更新架构信息的能力。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-139">Except in extreme cases, the account running the application doesn't need the ability to update schema information.</span></span> <span data-ttu-id="b9ce6-140">应该有多个帐户提供不同级别的权限。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-140">There should be several accounts that provide different levels of privilege.</span></span> <span data-ttu-id="b9ce6-141">应用程序应仅使用授予对表中数据的读取和写入访问权限的权限级别。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-141">The application should only use the permission level that grants read and writes access to the data in the tables.</span></span> <span data-ttu-id="b9ce6-142">这种保护将消除旨在丢弃数据库表或引入恶意触发器的攻击。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-142">This kind of protection would eliminate attacks that aimed to drop database tables or introduce malicious triggers.</span></span>

<span data-ttu-id="b9ce6-143">构建云本机应用程序的几乎每个部分都可以从记住最低权限原则中获益。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-143">Almost every part of building a cloud-native application can benefit from remembering the principle of least privilege.</span></span> <span data-ttu-id="b9ce6-144">在基于角色的访问控制中设置防火墙、网络安全组、角色和作用域时，可以在 (RBAC) 中找到它。</span><span class="sxs-lookup"><span data-stu-id="b9ce6-144">You can find it at play when setting up firewalls, network security groups, roles, and scopes in Role-based access control (RBAC).</span></span>

## <a name="securing-the-build"></a><span data-ttu-id="b9ce6-166">保护生成</span><span class="sxs-lookup"><span data-stu-id="b9ce6-166">Securing the build</span></span>
