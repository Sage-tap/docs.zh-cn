## <a name="rule-description"></a><span data-ttu-id="fe4ee-115">规则说明</span><span class="sxs-lookup"><span data-stu-id="fe4ee-115">Rule description</span></span>

<span data-ttu-id="fe4ee-116">.NET 5.0 增加了新属性，用于为特定于平台的 API 添加注释。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-116">.NET 5.0 added new attributes to annotate platform-specific APIs.</span></span> <span data-ttu-id="fe4ee-117">工作方式如下：</span><span class="sxs-lookup"><span data-stu-id="fe4ee-117">This works as follows:</span></span>

- <span data-ttu-id="fe4ee-118">将无标记的 API 视为适用于所有 OS 平台。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-118">An unmarked API is considered to work on all OS platforms.</span></span>
- <span data-ttu-id="fe4ee-119">将带有 `[SupportedOSPlatform("platformName")]` 标记的 API 视为仅可移植到指定的 OS 平台（该属性可以在不同的平台上应用多次）。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-119">An API marked with `[SupportedOSPlatform("platformName")]` is considered only portable to the specified OS platforms (the attribute can be applied multiple times with different platforms).</span></span>
- <span data-ttu-id="fe4ee-120">将带有 `[UnsupportedOSPlatform("platformName")]` 标记的 API 视为仅不支持指定的 OS 平台（该属性可以在不同的平台上应用多次）。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-120">An API marked with `[UnsupportedOSPlatform("platformName")]` is considered unsupported only to the specified OS platforms (the attribute can be applied multiple times with different platforms).</span></span>
- <span data-ttu-id="fe4ee-121">可以使用或不使用作为平台名称一部分的版本号对两个属性进行实例化。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-121">Both attributes can be instantiated with or without version numbers as part of the platform name.</span></span>
- <span data-ttu-id="fe4ee-122">如果存在 `[SupportedOSPlatform] and [UnsupportedOSPlatform]` 属性的组合，则按 OS 平台标识符将所有属性分组：</span><span class="sxs-lookup"><span data-stu-id="fe4ee-122">If a combination of `[SupportedOSPlatform] and [UnsupportedOSPlatform]` attributes are present, we group all attributes by OS platform identifier:</span></span>
  - <span data-ttu-id="fe4ee-123">允许列表。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-123">**Allow list**.</span></span> <span data-ttu-id="fe4ee-124">如果每个 OS 平台的最低版本是 `[SupportedOSPlatform]` 属性，则 API 会被视为仅在列出的平台上受支持，但在所有其他平台上不受支持。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-124">If the lowest version for each OS platform is a `[SupportedOSPlatform]` attribute, the API is considered to only be supported by the listed platforms and unsupported by all other platforms.</span></span> <span data-ttu-id="fe4ee-125">此列表可能具有 `[UnsupportedOSPlatform]` 属性，包含相同的平台，但版本较高，这表示 API 已从该版本中删除。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-125">The list could have `[UnsupportedOSPlatform]` attribute with same platform but only with higher version which denotes that the API is removed from that version.</span></span>
  - <span data-ttu-id="fe4ee-126">拒绝列表。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-126">**Deny list**.</span></span> <span data-ttu-id="fe4ee-127">如果每个 OS 平台的最低版本是 `[UnsupportedOSPlatform]` 属性，则 API 会被视为仅在列出的平台上不受支持，但在所有其他平台上受支持。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-127">If the lowest version for each OS platform is an `[UnsupportedOSPlatform]` attribute, then the API is considered to only be unsupported by the listed platforms and supported by all other platforms.</span></span> <span data-ttu-id="fe4ee-128">此列表可能具有 `[SupportedOSPlatform]` 属性，包含相同的平台，但版本较高，这表示该版本中已添加了对 API 的支持。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-128">The list could have `[SupportedOSPlatform]` attribute with same platform but only with higher version which denotes that the API is added support from that version.</span></span>
  - <span data-ttu-id="fe4ee-129">不一致的列表。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-129">**Inconsistent list**.</span></span> <span data-ttu-id="fe4ee-130">如果某些平台的最低版本是 `[SupportedOSPlatform]`，而其他平台的最低版本是 `[UnsupportedOSPlatform]`，则将此情况视为不一致，并忽略此 API 上的某些注释。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-130">If the lowest version for some platforms is `[SupportedOSPlatform]` while it is `[UnsupportedOSPlatform]` for other platforms, is considered inconsistent, and the some annotations on the API are ignored.</span></span> <span data-ttu-id="fe4ee-131">我们计划在未来引入另一个分析器，它将在遇到不一致时生成警告。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-131">We plan to introduce another analyzer producing a warning in case of inconsistency in the future.</span></span>

<span data-ttu-id="fe4ee-132">从不同平台的上下文中访问带有上述属性注释的 API 时，可能会造成违规。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-132">Accessing an APIs annotated with the above attributes from the context of a different platform could cause violations.</span></span>

### <a name="violations"></a><span data-ttu-id="fe4ee-133">冲突</span><span class="sxs-lookup"><span data-stu-id="fe4ee-133">Violations</span></span>

- <span data-ttu-id="fe4ee-134">从其他平台可访问的代码中访问仅在指定平台 (`[SupportedOSPlatform("platformName")]`) 上受支持的 API 时，将看到以下违规行为：“platformName”上支持“API”。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-134">If you access an API that's supported only on a specified platform (`[SupportedOSPlatform("platformName")]`) from code reachable on other platforms, you'll see the following violation: **'API' is supported on 'platformName'**.</span></span>

  ```csharp
  // An API supported only on Linux.
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { }

  // An API supported on Windows and iOS from version 14.0.
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }

  public void Caller()
  {
      LinuxOnlyApi(); // warns CA1416: 'LinuxOnlyApi' is supported on 'linux'

      SupportedOnWindowsAndIos14(); // warns CA1416:'SupportedOnWindowsAndIos14' is supported on 'windows'
                                    // warns CA1416: 'SupportedOnWindowsAndIos14' is supported on 'ios' 14.0 and later
  }

  ```

  > [!NOTE]
  > <span data-ttu-id="fe4ee-135">只有当项目未面向支持的平台 (`net5.0-differentPlatform`) 时，才会发生违规行为。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-135">A violation only occurs if the project does not target the supported platform (`net5.0-differentPlatform`).</span></span> <span data-ttu-id="fe4ee-136">这同样适用于多目标项目。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-136">This also applies to multi-targeted projects.</span></span> <span data-ttu-id="fe4ee-137">如果项目面向指定平台 (`net5.0-platformName`)，则不会发生违规行为。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-137">No violation occurs if the project targets the specified platform (`net5.0-platformName`).</span></span>

- <span data-ttu-id="fe4ee-138">从面向不受支持的平台上下文中访问具有 `[UnsupportedOSPlatform("platformName")]` 属性的 API 时，将发生以下违规行为：“platformName”上不支持“API”。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-138">Accessing an API that's attributed with `[UnsupportedOSPlatform("platformName")]` from a context that targets the unsupported platform could produce a violation: **'API' is unsupported on 'platformName'**.</span></span>

  ```csharp
  // An API not supported on Android but supported on all other platforms.
  [UnsupportedOSPlatform("android")]
  public void DoesNotWorkOnAndroid() { }

  // An API was unsupported on Windows until version 10.0.1903.
  // The API is considered supported everywhere else without constraints.
  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  public void Caller()
  {
      DoesNotWorkOnAndroid(); // warns CA1416:'DoesNotWorkOnAndroid' is unsupported on 'android'

      StartedWindowsSupportFromCertainVersion(); // warns CA1416:'StartedWindowsSupportFromCertainVersion' is unsupported on 'windows'
                                                 // warns CA1416:'StartedWindowsSupportFromCertainVersion' is supported on 'windows' 10.0.1903 and later
  }
  ```

  > [!NOTE]
  > <span data-ttu-id="fe4ee-139">如果你要生成的应用不以不受支持的平台为目标，则不会发生任何违规行为。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-139">If you're building an app that doesn't target the unsupported platform, you won't get any violations.</span></span> <span data-ttu-id="fe4ee-140">只有在以下情况下，才会发生违规行为：</span><span class="sxs-lookup"><span data-stu-id="fe4ee-140">A violation only occurs in the following cases:</span></span>
  >
  > - <span data-ttu-id="fe4ee-141">项目面向已被归为不受支持类别的平台。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-141">The project targets the platform attributed as unsupported.</span></span>
  > - <span data-ttu-id="fe4ee-142">`platformName` 包含在默认的 `MSBuild` `<SupportedPlatform>` 项目组中。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-142">The `platformName` is included in the default `MSBuild` `<SupportedPlatform>` items group.</span></span>
  > - <span data-ttu-id="fe4ee-143">`platformName` 通过手动方式包含在 MSBuild `<SupportedPlatform>` 项目组中。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-143">`platformName` is manually included in the MSBuild `<SupportedPlatform>` items group.</span></span>
  >
  >  ```XML
  >  <ItemGroup>
  >      <SupportedPlatform Include="platformName" />
  >  </ItemGroup>
  >  ```

## <a name="how-to-fix-violations"></a><span data-ttu-id="fe4ee-144">如何解决冲突</span><span class="sxs-lookup"><span data-stu-id="fe4ee-144">How to fix violations</span></span>

<span data-ttu-id="fe4ee-145">若要解决违规问题，建议确保只在相应的平台上运行时调用特定于平台的 API。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-145">The recommended way to deal with violations is to make sure you only call platform-specific APIs when running on an appropriate platform.</span></span> <span data-ttu-id="fe4ee-146">为此，请在生成时使用 `#if` 和多目标排除代码，或者在运行时有条件地调用代码。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-146">You can achieve this by excluding the code at build time using `#if` and multi-targeting, or by conditionally calling the code at run time.</span></span> <span data-ttu-id="fe4ee-147">分析器会识别 <xref:System.OperatingSystem> 类和 <xref:System.Runtime.InteropServices.RuntimeInformation.IsOSPlatform%2A?displayProperty=fullName> 中的平台保护。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-147">The analyzer recognizes the platform guards in the <xref:System.OperatingSystem> class and <xref:System.Runtime.InteropServices.RuntimeInformation.IsOSPlatform%2A?displayProperty=fullName>.</span></span>

- <span data-ttu-id="fe4ee-148">将调用站点囊括在平台保护方法的范围中，可抑制违规行为。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-148">Suppress violations by surrounding the call site with the platform guard methods.</span></span>

  ```csharp
  // An API supported only on Linux.
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { }

  // An API supported on Windows and iOS from version 14.0.
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }

  public void Caller()
  {
      LinuxOnlyApi(); // warns CA1416: 'LinuxOnlyApi' is supported on 'linux'

      SupportedOnWindowsAndIos14(); // warns CA1416:'SupportedOnWindowsAndIos14' is supported on 'windows'
                                    // warns CA1416: 'SupportedOnWindowsAndIos14' is supported on 'ios' 14.0 and later
  }

  // The warnings are avoided using platform guard methods.
  public void Caller()
  {
      if (OperatingSystem.IsLinux())
      {
          LinuxOnlyApi(); // no diagnostic
      }

      if (OperatingSystem.IsIOSVersionAtLeast(14))
      {
          SupportedOnWindowsAndIos14(); // no diagnostic
      }

      if(RuntimeInformation.IsOSPlatform(OSPlatform.Windows))
      {
          SupportedOnWindowsAndIos14(); // no diagnostic
      }
  }

  // An API not supported on Android but supported on all other platforms.
  [UnsupportedOSPlatform("android")]
  public void DoesNotWorkOnAndroid() { }

  // An API was unsupported on Windows until version 10.0.1903.
  // The API is considered supported everywhere else without constraints.
  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  public void Caller()
  {
      DoesNotWorkOnAndroid(); // warns CA1416:'DoesNotWorkOnAndroid' is unsupported on 'android'

      StartedWindowsSupportFromCertainVersion(); // warns CA1416:'StartedWindowsSupportFromCertainVersion' is unsupported on 'windows'
                                                 // warns CA1416:'StartedWindowsSupportFromCertainVersion' is supported on 'windows' 10.0.1903 and later
  }

  public void Caller()
  {
      if (!OperatingSystem.IsAndroid())
      {
          DoesNotWorkOnAndroid(); // no diagnostic
      }

      // Use the && and || logical operators to guard combined attributes.
      if (!OperatingSystem.IsWindows() || OperatingSystem.IsWindowsVersionAtLeast(10, 0, 1903))
      {
          StartedWindowsSupportFromCertainVersion(); // no diagnostic
      }
  }

  ```

- <span data-ttu-id="fe4ee-149">分析器还会使用 <xref:System.Diagnostics.Debug.Assert%2A?displayProperty=fullName> 来防止不受支持的平台访问代码。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-149">The analyzer also respects <xref:System.Diagnostics.Debug.Assert%2A?displayProperty=fullName> as a means for preventing the code from being reached on unsupported platforms.</span></span> <span data-ttu-id="fe4ee-150">使用 `Debug.Assert` 时，可以根据需要从发行版本中去除此检查。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-150">Using `Debug.Assert` allows the check to be trimmed out of release builds, if desired.</span></span>

  ```csharp
  // An API supported only on Linux.
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { }

  public void Caller()
  {
      Debug.Assert(OperatingSystem.IsLinux());

      LinuxOnlyApi(); // No diagnostic
  }
  ```

- <span data-ttu-id="fe4ee-151">可以选择将自己的 API 标记为特定于平台，从而有效地将要求转发给调用方。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-151">You can choose to mark your own APIs as being platform-specific, effectively forwarding the requirements to your callers.</span></span> <span data-ttu-id="fe4ee-152">可以将平台属性应用于以下任何 API：</span><span class="sxs-lookup"><span data-stu-id="fe4ee-152">You can apply platform attributes to any of the following APIs:</span></span>

  - <span data-ttu-id="fe4ee-153">类型</span><span class="sxs-lookup"><span data-stu-id="fe4ee-153">Types</span></span>
  - <span data-ttu-id="fe4ee-154">成员（方法、字段、属性和事件）</span><span class="sxs-lookup"><span data-stu-id="fe4ee-154">Members (methods, fields, properties, and events)</span></span>
  - <span data-ttu-id="fe4ee-155">程序集</span><span class="sxs-lookup"><span data-stu-id="fe4ee-155">Assemblies</span></span>

  ```csharp
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }

  [SupportedOSPlatform("ios15.0")] // call site version should be equal to or higher than the API version
  public void Caller()
  {
      SupportedOnWindowsAndIos14(); // No diagnostics
  }

  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void Caller()
  {
      StartedWindowsSupportFromCertainVersion(); // No diagnostics
  }
  ```

- <span data-ttu-id="fe4ee-156">在应用程序集或类型级别的属性时，程序集或类型中的所有成员都被视为特定于平台。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-156">When an assembly or type-level attribute is applied, all members within the assembly or type are considered to be platform specific.</span></span>

  ```csharp
  [assembly:SupportedOSPlatform("windows")]
  public namespace ns
  {
      public class Sample
      {
          public void SupportedOnWindows() { }

          public void Caller()
          {
              SupportedOnWindows(); // No diagnostic as call site and calling method both windows only
          }
      }
  }
  ```

## <a name="when-to-suppress-warnings"></a><span data-ttu-id="fe4ee-157">何时禁止显示警告</span><span class="sxs-lookup"><span data-stu-id="fe4ee-157">When to suppress warnings</span></span>

<span data-ttu-id="fe4ee-158">在没有适当平台上下文或不受保护的情况下，不建议引用特定于平台的 API。</span><span class="sxs-lookup"><span data-stu-id="fe4ee-158">Referencing platform-specific APIs without a proper platform context or guard is not recommended.</span></span> <span data-ttu-id="fe4ee-159">不过，你可以通过常规方式（`<NoWarn>`、.editorconfig 文件或 `#pragma`）抑制这些诊断：</span><span class="sxs-lookup"><span data-stu-id="fe4ee-159">However, you can suppress these diagnostics by the usual means (`<NoWarn>`, .editorconfig file, or `#pragma`):</span></span>

```csharp
public void Caller()
{
#pragma warning disable CA1416
    LinuxOnlyApi();
#pragma warning restore CA1416
}
```
