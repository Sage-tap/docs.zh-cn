<span data-ttu-id="2f0d8-167">与大多数对象模型一样，某些设计模式适用于 `My` 命名空间，而另一些设计模式则不适用。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-167">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="2f0d8-168">设计 `My` 命名空间扩展时，请考虑以下原则：</span><span class="sxs-lookup"><span data-stu-id="2f0d8-168">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="2f0d8-169">**无状态方法。**</span><span class="sxs-lookup"><span data-stu-id="2f0d8-169">**Stateless methods.**</span></span> <span data-ttu-id="2f0d8-170">`My` 命名空间中的方法应提供针对特定任务的完整解决方案。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-170">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="2f0d8-171">确保传递给方法的参数值提供完成特定任务所需的全部输入。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-171">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="2f0d8-172">避免创建依赖于之前状态的方法，例如与资源的开放连接。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-172">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>
- <span data-ttu-id="2f0d8-173">**全局实例。**</span><span class="sxs-lookup"><span data-stu-id="2f0d8-173">**Global instances.**</span></span> <span data-ttu-id="2f0d8-174">`My` 命名空间中维护的唯一状态是项目的全局状态。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-174">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="2f0d8-175">例如，`My.Application.Info` 封装整个应用程序共享的状态。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-175">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>
- <span data-ttu-id="2f0d8-176">**简单参数类型。**</span><span class="sxs-lookup"><span data-stu-id="2f0d8-176">**Simple parameter types.**</span></span> <span data-ttu-id="2f0d8-177">通过避免使用复杂参数类型使操作变得简单。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-177">Keep things simple by avoiding complex parameter types.</span></span> <span data-ttu-id="2f0d8-178">相反，创建不使用参数输入或使用简单输入类型（例如字符串、基元类型等）的方法。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-178">Instead, create methods that either take no parameter input or that take simple input types such as strings, primitive types, and so on.</span></span>
- <span data-ttu-id="2f0d8-179">**工厂方法。**</span><span class="sxs-lookup"><span data-stu-id="2f0d8-179">**Factory methods.**</span></span> <span data-ttu-id="2f0d8-180">某些类型必然很难实例化。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-180">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="2f0d8-181">提供工厂方法作为 `My` 命名空间的扩展，这样一来，你可以更轻松地发现和使用属于此类别的类型。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-181">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="2f0d8-182">`My.Computer.FileSystem.OpenTextFileReader` 是一个效果很好的工厂方法示例。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-182">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="2f0d8-183">.NET Framework 中有几种可用的流类型。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-183">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="2f0d8-184">通过专门指定文本文件，`OpenTextFileReader` 可帮助用户了解要使用的流。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-184">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>

<span data-ttu-id="2f0d8-185">这些指南不妨碍类库的一般设计原则。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-185">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="2f0d8-186">相反，它们是针对正在使用 Visual Basic 和 `My` 命名空间的开发人员进行优化的建议。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-186">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="2f0d8-187">有关创建类库的一般设计原则，请参阅[框架设计指南](../../../standard/design-guidelines/index.md)。</span><span class="sxs-lookup"><span data-stu-id="2f0d8-187">For general design principles for creating class libraries, see [Framework Design Guidelines](../../../standard/design-guidelines/index.md).</span></span>
