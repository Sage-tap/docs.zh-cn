# <a name="instance-activation"></a><span data-ttu-id="95bc8-103">实例激活</span><span class="sxs-lookup"><span data-stu-id="95bc8-103">Instance Activation</span></span>

<span data-ttu-id="95bc8-104">SQL 工作流实例存储运行一个内部任务，该任务将定期唤醒并检测持久性数据库中是否有可运行或可激活的工作流实例。</span><span class="sxs-lookup"><span data-stu-id="95bc8-104">The SQL Workflow Instance Store runs an internal task that periodically wakes up and detects runnable or activatable workflow instances in the persistence database.</span></span> <span data-ttu-id="95bc8-105">如果找到可运行的工作流实例，SQL 工作流实例存储将通知能够激活该实例的工作流主机。</span><span class="sxs-lookup"><span data-stu-id="95bc8-105">If it finds a runnable workflow instance, it notifies the workflow host that is capable of activating the instance.</span></span> <span data-ttu-id="95bc8-106">如果实例存储找到可激活的工作流实例，它将通知用于激活工作流主机的泛型主机，而泛型主机将运行此工作流实例。</span><span class="sxs-lookup"><span data-stu-id="95bc8-106">If the instance store finds an activatable workflow instance, it notifies a generic host that activates a workflow host, which in turn runs the workflow instance.</span></span> <span data-ttu-id="95bc8-107">本主题的以下各节详细说明了实例激活过程。</span><span class="sxs-lookup"><span data-stu-id="95bc8-107">The following sections in this topic explain the instance activation process in detail.</span></span>  
  
## <a name="detecting-and-activating-runnable-workflow-instances"></a><a name="RunnableSection"></a> <span data-ttu-id="95bc8-108">检测和激活可运行的工作流实例</span><span class="sxs-lookup"><span data-stu-id="95bc8-108">Detecting and Activating Runnable Workflow Instances</span></span>  

 <span data-ttu-id="95bc8-109">如果实例未处于挂起状态或完成状态并且满足以下条件，则 SQL 工作流实例存储会将工作流实例视为可 *运行* 状态：</span><span class="sxs-lookup"><span data-stu-id="95bc8-109">The SQL Workflow Instance Store considers a workflow instance *runnable* if the instance is not in the suspended state or the completed state and satisfies the following conditions:</span></span>  
  
- <span data-ttu-id="95bc8-110">实例处于解除锁定状态，并且具有已过期的挂起计时器。</span><span class="sxs-lookup"><span data-stu-id="95bc8-110">The instance is unlocked and has a pending timer that has expired.</span></span>  
  
- <span data-ttu-id="95bc8-111">实例上的锁已过期。</span><span class="sxs-lookup"><span data-stu-id="95bc8-111">The instance has an expired lock on it.</span></span>  
  
- <span data-ttu-id="95bc8-112">实例处于解除锁定状态，并且其状态为 "正在 **执行**"。</span><span class="sxs-lookup"><span data-stu-id="95bc8-112">The instance is unlocked and its status is **Executing**.</span></span>  
  
 <span data-ttu-id="95bc8-113">当 SQL 工作流实例存储找到可运行的实例时，将引发 <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent>。</span><span class="sxs-lookup"><span data-stu-id="95bc8-113">The SQL Workflow Instance Store raises the <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> when it finds a runnable instance.</span></span> <span data-ttu-id="95bc8-114">此后，SqlWorkflowInstanceStore 将停止监视，直到在该存储上调用一次 <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand>。</span><span class="sxs-lookup"><span data-stu-id="95bc8-114">After this, the SqlWorkflowInstanceStore stops monitoring until the <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> is called once on the store.</span></span>  
  
 <span data-ttu-id="95bc8-115">已订阅 <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> 且能够加载该实例的工作流主机将针对实例存储执行 <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand>，以将该实例加载到内存中。</span><span class="sxs-lookup"><span data-stu-id="95bc8-115">A workflow host that has subscribed for the <xref:System.Activities.DurableInstancing.HasRunnableWorkflowEvent> and capable of loading the instance executes the <xref:System.Activities.DurableInstancing.TryLoadRunnableWorkflowCommand> against the instance store to load the instance into memory.</span></span> <span data-ttu-id="95bc8-116">如果主机和实例的元数据属性 **WorkflowServiceType** 设置为相同的值，则认为工作流主机可以加载工作流实例。</span><span class="sxs-lookup"><span data-stu-id="95bc8-116">A workflow host is considered capable of loading a workflow instance if the host and the instance have metadata property **WorkflowServiceType** set to the same value.</span></span>  
  
## <a name="detecting-and-activating-activatable-workflow-instances"></a><span data-ttu-id="95bc8-117">检测和激活可激活的工作流实例</span><span class="sxs-lookup"><span data-stu-id="95bc8-117">Detecting and Activating Activatable Workflow Instances</span></span>  

 <span data-ttu-id="95bc8-118">如果实例可运行，并且没有能够加载该实例的工作流主机正在计算机上运行，则该工作流实例将被视为可 *激活* 。</span><span class="sxs-lookup"><span data-stu-id="95bc8-118">A workflow instance is considered *activatable* if the instance is runnable and there is no workflow host that is capable of loading the instance is running on the computer.</span></span> <span data-ttu-id="95bc8-119">有关可运行的工作流实例的定义，请参见上面的“检测和激活可运行的工作流实例”。</span><span class="sxs-lookup"><span data-stu-id="95bc8-119">See Detecting and Activating Runnable Workflow Instances above for the definition of a runnable workflow instance.</span></span>  
  
 <span data-ttu-id="95bc8-120">当 SQL 工作流实例存储在数据库中找到可激活的工作流实例时，将引发 <xref:System.Activities.DurableInstancing.HasActivatableWorkflowEvent>。</span><span class="sxs-lookup"><span data-stu-id="95bc8-120">The SQL Workflow Instance Store raises the <xref:System.Activities.DurableInstancing.HasActivatableWorkflowEvent> when it finds an activatable workflow instance in the database.</span></span> <span data-ttu-id="95bc8-121">此后，SqlWorkflowInstanceStore 将停止监视，直到在该存储上调用一次 <xref:System.Activities.DurableInstancing.QueryActivatableWorkflowsCommand>。</span><span class="sxs-lookup"><span data-stu-id="95bc8-121">After this, the SqlWorkflowInstanceStore stops monitoring until the <xref:System.Activities.DurableInstancing.QueryActivatableWorkflowsCommand> is called once on the store.</span></span>  
  
 <span data-ttu-id="95bc8-122">当已订阅 <xref:System.Activities.DurableInstancing.HasActivatableWorkflowEvent> 的泛型主机接收到此事件时，它将针对实例存储执行 <xref:System.Activities.DurableInstancing.QueryActivatableWorkflowsCommand> 以获取创建工作流主机所需的激活参数。</span><span class="sxs-lookup"><span data-stu-id="95bc8-122">When a generic host that has subscribed for the <xref:System.Activities.DurableInstancing.HasActivatableWorkflowEvent> receives the event, it executes the <xref:System.Activities.DurableInstancing.QueryActivatableWorkflowsCommand> against the instance store to obtain activation parameters required to create a workflow host.</span></span> <span data-ttu-id="95bc8-123">泛型主机使用这些激活参数来创建工作流主机，后者又依次加载并运行可运行的服务实例。</span><span class="sxs-lookup"><span data-stu-id="95bc8-123">The generic host uses these activation parameters to create a workflow host, which in turn loads and runs the runnable service instance.</span></span>  
  
## <a name="generic-hosts"></a><span data-ttu-id="95bc8-124">泛型主机</span><span class="sxs-lookup"><span data-stu-id="95bc8-124">Generic Hosts</span></span>

 <span data-ttu-id="95bc8-127">目前，SQL 工作流实例存储支持将 ActivationType 参数值设置为 **WAS** 的泛型主机。</span><span class="sxs-lookup"><span data-stu-id="95bc8-127">Currently, the SQL Workflow Instance Store supports generic hosts with value of the ActivationType parameter set to **WAS**.</span></span> <span data-ttu-id="95bc8-128">如果未将 ActivationType 设置为 WAS，SQL 工作流实例存储将引发 <xref:System.Runtime.DurableInstancing.InstancePersistenceException>。</span><span class="sxs-lookup"><span data-stu-id="95bc8-128">If the ActivationType is not set to WAS, the SQL Workflow Instance Store throws an <xref:System.Runtime.DurableInstancing.InstancePersistenceException>.</span></span> <span data-ttu-id="95bc8-129">Windows Server AppFabric 的承载功能附带的工作流管理服务是将激活类型设置为 **WAS** 的泛型主机。</span><span class="sxs-lookup"><span data-stu-id="95bc8-129">The Workflow Management Service that ships with the hosting features of Windows Server AppFabric is a generic host that has the activation type set to **WAS**.</span></span>  
  
 <span data-ttu-id="95bc8-130">为了实现 WAS 激活，泛型主机需要一组激活参数来派生激活新主机所在的终结点地址。</span><span class="sxs-lookup"><span data-stu-id="95bc8-130">For WAS activation, a generic host requires a set of activation parameters to derive the endpoint address at which new hosts can be activated.</span></span> <span data-ttu-id="95bc8-131">用于 WAS 激活的激活参数包括：站点名称、应用程序路径（相对于站点）和服务路径（相对于应用程序）。</span><span class="sxs-lookup"><span data-stu-id="95bc8-131">The activation parameters for WAS activation are name of the site, path to the application relative to the site, and path to the service relative to the application.</span></span> <span data-ttu-id="95bc8-132">SQL 工作流实例存储在执行 <xref:System.Activities.DurableInstancing.SaveWorkflowCommand> 期间存储这些激活参数。</span><span class="sxs-lookup"><span data-stu-id="95bc8-132">The SQL Workflow Instance Store stores these activation parameters during the execution of the <xref:System.Activities.DurableInstancing.SaveWorkflowCommand>.</span></span>
