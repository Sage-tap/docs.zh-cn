# <a name="bc42358-because-this-call-is-not-awaited-execution-of-the-current-method-continues-before-the-call-is-completed"></a><span data-ttu-id="b3b27-103">BC42358：由于不等待此调用，因此在调用完成之前，当前方法的执行将继续</span><span class="sxs-lookup"><span data-stu-id="b3b27-103">BC42358: Because this call is not awaited, execution of the current method continues before the call is completed</span></span>

<span data-ttu-id="b3b27-106">当前方法调用返回 <xref:System.Threading.Tasks.Task> 或的异步方法 <xref:System.Threading.Tasks.Task%601> ，而不将 [Await](../operators/await-operator.md) 运算符应用于结果。</span><span class="sxs-lookup"><span data-stu-id="b3b27-106">The current method calls an async method that returns a <xref:System.Threading.Tasks.Task> or a <xref:System.Threading.Tasks.Task%601> and doesn't apply the [Await](../operators/await-operator.md) operator to the result.</span></span> <span data-ttu-id="b3b27-107">对异步方法的调用会启动异步任务。</span><span class="sxs-lookup"><span data-stu-id="b3b27-107">The call to the async method starts an asynchronous task.</span></span> <span data-ttu-id="b3b27-108">但是，由于没有应用 `Await` 运算符，程序将继续运行，而不必等待任务完成。</span><span class="sxs-lookup"><span data-stu-id="b3b27-108">However, because no `Await` operator is applied, the program continues without waiting for the task to complete.</span></span> <span data-ttu-id="b3b27-109">在大多数情况下，此行为并不是你所期望的。</span><span class="sxs-lookup"><span data-stu-id="b3b27-109">In most cases, that behavior isn't expected.</span></span> <span data-ttu-id="b3b27-110">方法调用的其他方面通常取决于调用的结果，而调用的方法至少应在从包含调用的方法返回之前完成。</span><span class="sxs-lookup"><span data-stu-id="b3b27-110">Usually other aspects of the calling method depend on the results of the call or, minimally, the called method is expected to complete before you return from the method that contains the call.</span></span>

<span data-ttu-id="b3b27-111">对于被调用的异步方法中引发的异常，如何进行处理同样是一个重要的问题。</span><span class="sxs-lookup"><span data-stu-id="b3b27-111">An equally important issue is what happens with exceptions that are raised in the called async method.</span></span> <span data-ttu-id="b3b27-112">在返回 <xref:System.Threading.Tasks.Task> 或 <xref:System.Threading.Tasks.Task%601> 的方法中引发的异常将存储在返回的任务中。</span><span class="sxs-lookup"><span data-stu-id="b3b27-112">An exception that's raised in a method that returns a <xref:System.Threading.Tasks.Task> or  <xref:System.Threading.Tasks.Task%601> is stored in the returned task.</span></span> <span data-ttu-id="b3b27-113">如果不等待任务或不显式检查异常，该异常会丢失。</span><span class="sxs-lookup"><span data-stu-id="b3b27-113">If you don't await the task or explicitly check for exceptions, the exception is lost.</span></span> <span data-ttu-id="b3b27-114">如果等待任务，将重新引发异常。</span><span class="sxs-lookup"><span data-stu-id="b3b27-114">If you await the task, its exception is rethrown.</span></span>

## <a name="to-address-this-warning"></a><span data-ttu-id="b3b27-119">解决此警告</span><span class="sxs-lookup"><span data-stu-id="b3b27-119">To address this warning</span></span>

<span data-ttu-id="b3b27-120">只有当您确定不需要等待异步调用完成并且调用的方法不会引发任何异常时，才应考虑禁止显示警告。</span><span class="sxs-lookup"><span data-stu-id="b3b27-120">You should consider suppressing the warning only if you're sure that you don't want to wait for the asynchronous call to complete and that the called method won't raise any exceptions.</span></span> <span data-ttu-id="b3b27-121">在此情况下，可以通过将调用的任务结果分配给变量来禁止显示警告。</span><span class="sxs-lookup"><span data-stu-id="b3b27-121">In that case, you can suppress the warning by assigning the task result of the call to a variable.</span></span>

<span data-ttu-id="b3b27-122">下面的示例演示如何导致此警告，如何禁止显示该警告，以及如何等待调用：</span><span class="sxs-lookup"><span data-stu-id="b3b27-122">The following example shows how to cause the warning, how to suppress it, and how to await the call:</span></span>

```vb
Async Function CallingMethodAsync() As Task

    ResultsTextBox.Text &= vbCrLf & "  Entering calling method."

    ' Variable delay is used to slow down the called method so that you
    ' can distinguish between awaiting and not awaiting in the program's output.
    ' You can adjust the value to produce the output that this topic shows
    ' after the code.
    Dim delay = 5000

    ' Call #1.
    ' Call an async method. Because you don't await it, its completion isn't
    ' coordinated with the current method, CallingMethodAsync.
    ' The following line causes the warning.
    CalledMethodAsync(delay)

    ' Call #2.
    ' To suppress the warning without awaiting, you can assign the
    ' returned task to a variable. The assignment doesn't change how
    ' the program runs. However, the recommended practice is always to
    ' await a call to an async method.
    ' Replace Call #1 with the following line.
    'Task delayTask = CalledMethodAsync(delay)

    ' Call #3
    ' To contrast with an awaited call, replace the unawaited call
    ' (Call #1 or Call #2) with the following awaited call. The best
    ' practice is to await the call.

    'Await CalledMethodAsync(delay)

    ' If the call to CalledMethodAsync isn't awaited, CallingMethodAsync
    ' continues to run and, in this example, finishes its work and returns
    ' to its caller.
    ResultsTextBox.Text &= vbCrLf & "  Returning from calling method."
End Function

Async Function CalledMethodAsync(howLong As Integer) As Task

    ResultsTextBox.Text &= vbCrLf & "    Entering called method, starting and awaiting Task.Delay."
    ' Slow the process down a little so you can distinguish between awaiting
    ' and not awaiting. Adjust the value for howLong if necessary.
    Await Task.Delay(howLong)
    ResultsTextBox.Text &= vbCrLf & "    Task.Delay is finished--returning from called method."
End Function
```

<span data-ttu-id="b3b27-123">在此示例中，如果选择调用 1 或调用 2，未等待的异步方法 (`CalledMethodAsync`) 将在调用方 (`CallingMethodAsync`) 和调用方的调用方 (`StartButton_Click`) 完成后完成。</span><span class="sxs-lookup"><span data-stu-id="b3b27-123">In the example, if you choose Call #1 or Call #2, the unawaited async method (`CalledMethodAsync`) finishes after both its caller (`CallingMethodAsync`) and the caller's caller (`StartButton_Click`) are complete.</span></span> <span data-ttu-id="b3b27-124">以下输出的最后一行演示了调用的方法何时完成。</span><span class="sxs-lookup"><span data-stu-id="b3b27-124">The last line in the following output shows you when the called method finishes.</span></span> <span data-ttu-id="b3b27-125">输出内容标示了整个示例中何时进入和退出调用 `CallingMethodAsync` 的事件处理程序。</span><span class="sxs-lookup"><span data-stu-id="b3b27-125">Entry to and exit from the event handler that calls `CallingMethodAsync` in the full example are marked in the output.</span></span>

```console
Entering the Click event handler.
  Entering calling method.
    Entering called method, starting and awaiting Task.Delay.
  Returning from calling method.
Exiting the Click event handler.
    Task.Delay is finished--returning from called method.
```
