## <a name="overview"></a><span data-ttu-id="566f3-110">概述</span><span class="sxs-lookup"><span data-stu-id="566f3-110">Overview</span></span>  

 <span data-ttu-id="566f3-111">由生成的代码 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 默认提供延迟加载功能。</span><span class="sxs-lookup"><span data-stu-id="566f3-111">The code generated by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] provides deferred loading capabilities by default.</span></span> <span data-ttu-id="566f3-112">延迟加载对在中间层以透明方式根据需要加载数据而言非常方便。</span><span class="sxs-lookup"><span data-stu-id="566f3-112">Deferred loading is very convenient on the mid-tier for transparent loading of data on demand.</span></span> <span data-ttu-id="566f3-113">但是，它会给序列化过程带来问题，原因是不论是否需要进行延迟加载，序列化程序都会触发延迟加载。</span><span class="sxs-lookup"><span data-stu-id="566f3-113">However, it is problematic for serialization, because the serializer triggers deferred loading whether deferred loading is intended or not.</span></span> <span data-ttu-id="566f3-114">实际上，对对象进行序列化时，会对其在所有出站延迟加载引用下的可传递闭包进行序列化。</span><span class="sxs-lookup"><span data-stu-id="566f3-114">In effect, when an object is serialized, its transitive closure under all outbound defer-loaded references is serialized.</span></span>  
  
 <span data-ttu-id="566f3-115">[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 的序列化功能主要通过以下两种机制来解决此问题：</span><span class="sxs-lookup"><span data-stu-id="566f3-115">The [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] serialization feature addresses this problem, primarily through two mechanisms:</span></span>  
  
- <span data-ttu-id="566f3-116">用于关闭延迟加载的 <xref:System.Data.Linq.DataContext> 模式 (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>)。</span><span class="sxs-lookup"><span data-stu-id="566f3-116">A <xref:System.Data.Linq.DataContext> mode for turning off deferred loading (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>).</span></span> <span data-ttu-id="566f3-117">有关详细信息，请参阅 <xref:System.Data.Linq.DataContext>。</span><span class="sxs-lookup"><span data-stu-id="566f3-117">For more information, see <xref:System.Data.Linq.DataContext>.</span></span>  
  
- <span data-ttu-id="566f3-118">用于在生成的实体中生成 <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> 和 <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> 属性的代码生成开关。</span><span class="sxs-lookup"><span data-stu-id="566f3-118">A code-generation switch to generate <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> and <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> attributes on generated entities.</span></span> <span data-ttu-id="566f3-119">这方面（包括处于序列化过程中的延迟加载类的行为）是本主题的主要介绍对象。</span><span class="sxs-lookup"><span data-stu-id="566f3-119">This aspect, including the behavior of defer-loading classes under serialization, is the major subject of this topic.</span></span>
