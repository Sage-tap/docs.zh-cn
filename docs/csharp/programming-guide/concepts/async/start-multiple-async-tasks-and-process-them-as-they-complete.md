## <a name="create-the-asynchronous-sum-page-sizes-method"></a><span data-ttu-id="63b37-125">创建异步总和页面大小方法</span><span class="sxs-lookup"><span data-stu-id="63b37-125">Create the asynchronous sum page sizes method</span></span>

<span data-ttu-id="63b37-126">在 `Main` 方法下，添加 `SumPageSizesAsync` 方法：</span><span class="sxs-lookup"><span data-stu-id="63b37-126">Below the `Main` method, add the `SumPageSizesAsync` method:</span></span>

```csharp
static async Task SumPageSizesAsync()
{
    var stopwatch = Stopwatch.StartNew();

    IEnumerable<Task<int>> downloadTasksQuery =
        from url in s_urlList
        select ProcessUrlAsync(url, s_client);

    List<Task<int>> downloadTasks = downloadTasksQuery.ToList();

    int total = 0;
    while (downloadTasks.Any())
    {
        Task<int> finishedTask = await Task.WhenAny(downloadTasks);
        downloadTasks.Remove(finishedTask);
        total += await finishedTask;
    }

    stopwatch.Stop();

    Console.WriteLine($"\nTotal bytes returned:  {total:#,#}");
    Console.WriteLine($"Elapsed time:          {stopwatch.Elapsed}\n");
}
```

<span data-ttu-id="63b37-127">该方法从实例化和启动 <xref:System.Diagnostics.Stopwatch> 开始。</span><span class="sxs-lookup"><span data-stu-id="63b37-127">The method starts by instantiating and starting a <xref:System.Diagnostics.Stopwatch>.</span></span> <span data-ttu-id="63b37-128">然后它会包含一个查询，执行此查询时，将创建任务集合。</span><span class="sxs-lookup"><span data-stu-id="63b37-128">It then includes a query that, when executed, creates a collection of tasks.</span></span> <span data-ttu-id="63b37-129">每次对以下代码中的 `ProcessUrlAsync` 进行调用都会返回 <xref:System.Threading.Tasks.Task%601>，其中 `TResult` 是一个整数：</span><span class="sxs-lookup"><span data-stu-id="63b37-129">Each call to `ProcessUrlAsync` in the following code returns a <xref:System.Threading.Tasks.Task%601>, where `TResult` is an integer:</span></span>

```csharp

    from url in s_urlList
    select ProcessUrlAsync(url, s_client);
```

<span data-ttu-id="63b37-130">由于 LINQ 的[延迟执行](../../../../standard/linq/deferred-execution-example.md)，因此可调用 <xref:System.Linq.Enumerable.ToList%2A?displayProperty=nameWithType> 来启动每个任务。</span><span class="sxs-lookup"><span data-stu-id="63b37-130">Due to [deferred execution](../../../../standard/linq/deferred-execution-example.md) with the LINQ, you call <xref:System.Linq.Enumerable.ToList%2A?displayProperty=nameWithType> to start each task.</span></span>


<span data-ttu-id="63b37-131">`while` 循环针对集合中的每个任务执行以下步骤：</span><span class="sxs-lookup"><span data-stu-id="63b37-131">The `while` loop performs the following steps for each task in the collection:</span></span>

1. <span data-ttu-id="63b37-132">等待调用 `WhenAny`，以标识集合中首个已完成下载的任务。</span><span class="sxs-lookup"><span data-stu-id="63b37-132">Awaits a call to `WhenAny` to identify the first task in the collection that has finished its download.</span></span>

1. <span data-ttu-id="63b37-133">从集合中移除任务。</span><span class="sxs-lookup"><span data-stu-id="63b37-133">Removes that task from the collection.</span></span>

    ```csharp

    ```

1. <span data-ttu-id="63b37-134">等待 `finishedTask`，由对 `ProcessUrlAsync` 的调用返回。</span><span class="sxs-lookup"><span data-stu-id="63b37-134">Awaits `finishedTask`, which is returned by a call to `ProcessUrlAsync`.</span></span> <span data-ttu-id="63b37-135">`finishedTask` 变量是 <xref:System.Threading.Tasks.Task%601>，其中 `TResult` 是整数。</span><span class="sxs-lookup"><span data-stu-id="63b37-135">The `finishedTask` variable is a <xref:System.Threading.Tasks.Task%601> where `TResult` is an integer.</span></span> <span data-ttu-id="63b37-136">任务已完成，但需等待它检索已下载网站的长度，如以下示例所示。</span><span class="sxs-lookup"><span data-stu-id="63b37-136">The task is already complete, but you await it to retrieve the length of the downloaded website, as the following example shows.</span></span> <span data-ttu-id="63b37-137">如果任务出错，`await` 将引发存储在 `AggregateException` 中的第一个子异常，这一点与读取 <xref:System.Threading.Tasks.Task%601.Result?displayProperty=nameWithType> 属性将引发 `AggregateException` 不同。</span><span class="sxs-lookup"><span data-stu-id="63b37-137">If the task is faulted, `await` will throw the first child exception stored in the `AggregateException`, unlike reading the <xref:System.Threading.Tasks.Task%601.Result?displayProperty=nameWithType> property, which would throw the `AggregateException`.</span></span>
